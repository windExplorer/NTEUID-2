# 角色评分与伤害计算说明

本文档说明本项目（NTEUID 插件）中「角色养成度评分」与「角色伤害」两套计算逻辑的实现方式、公式与数据来源。

- 评分相关代码：`NTEUID/nte_role/score.py` + 资源仓库 `NTEUID/resource/scoring/*`
- 伤害相关代码：`NTEUID/utils/damage/*`（`formula.py` / `profiles.py` / `models.py` / `buffs.py` / `constants.py`）

---

## 一、角色评分（养成度评分）

评分衡量某个角色在当前装备（核心 + 驱动）下，词条契合度与养成上限。完全基于资源仓库里的静态配置，不依赖敌人/增益。

### 1.1 数据来源（资源仓库 `scoring/`）

| 文件 | 作用 |
| --- | --- |
| `scoring/attributes.json` | 每条属性的「评分权重」`score`，以及是否是百分比 `is_percent` |
| `scoring/grades.json` | 评级阈值（按 `score / max_score` 的比值划分） |
| `scoring/chars/<char_id>.json` | 单角色评分方案：核心主词条、推荐副词条、各项上限 |

角色方案（`chars/<id>.json`）示例字段：

```json
{
  "id": "1003",
  "refer_score": 84,
  "core_main_attr_list": ["damageupincantationbase", "atkup"],
  "recommend_attr_list": ["damageupincantationbase", "atkup", "atkadd", "damageupgeneralbase"],
  "max": {
    "core": 55.0,
    "pie": { "2": 5.0, "3": 7.5, "4": 10.0625 },
    "raw_sum": 105.3125,
    "score": 126
  }
}
```

- `refer_score`：单件/整体评分的「参考分」（`raw_score` 折算成百分制的分母）。
- `core_main_attr_list`：核心（core）装备应当出的**主词条**（只有 `%` 结尾的主词条才计分）。
- `recommend_attr_list`：驱动（pie）装备应当刷的**推荐副词条**。
- `max.core` / `max.pie` / `max.score`：核心、各驱动槽位、整体角色评分的满分上限，用于评级。

### 1.2 单条属性的得分 `_prop_score`

```
解析值：
  若 value 以 "%" 结尾 → (数值/100, 百分比=True)
  否则          → (数值,    百分比=False)

weight = attributes[prop.id].score
若 weight <= 0 → 0（该属性不参与评分）
否则：
  得分 = (数值/100  若 百分比   否则  数值) / weight
```

即：百分比属用「百分比数字 ÷ 100」再除以权重；固定值属性直接用「数值」除以权重。权重越大，说明该属性越“贵”，同样数值得分越低。

### 1.3 单件装备计分 `score_equipment`

- **主词条（main_properties）**：仅当 `value` 以 `%` 结尾 **且** `prop.id ∈ core_main_attrs` 才计入（`CharacterScore.is_main_prop_counted`）。
- **副词条（properties）**：仅当 `index < item.lev // 5`（已解锁的副词条数 = 强化等级 ÷ 5）**且** `prop.id ∈ recommend_attrs` 才计入（`CharacterScore.is_sub_prop_counted`）。
- 单件 `raw_score` = 上述命中属性的 `_prop_score` 之和。
- 单件 `score` = `raw_score × 100 / refer_score`（百分制）。
- 单件评级 = `grade_of(raw_score / max_for(item))`：
  - 核心装备上限 = `max.core`；
  - 驱动装备（id 以 `cell` 开头）按槽位取 `max.pie[grid]`（grid 来自 id 前缀数字）。

### 1.4 角色总分与评级

```
raw_score  = Σ 各装备 raw_score
score_raw  = raw_score × 100 / refer_score
score      = ceil(score_raw)            # 向上取整
grade      = grade_of(score / max_score)
```

评级阈值（`grades.json`，按 `score / max_score` 比值从高到低匹配）：

| 字段 | 值 | 含义 |
| --- | --- | --- |
| S | `min_ratio = 0.8` | 比值 ≥ 0.8 |
| A | `min_ratio = 0.6` | 比值 ≥ 0.6 |
| B | `min_ratio = 0.0` | 其余 |

> 注意：`score`（展示分）按 `refer_score` 折算，而**评级**按 `max_score` 折算，两者分母不同——评级反映的是“相对该角色满分上限的达成度”，分是“相对参考分的养成度”。

### 1.5 入口与展示

- 核心函数：`score_character(character) -> CharacterScore | None`（`score.py`）。无方案（`chars/<id>.json` 不存在）或无装备时返回 `None`。
- 展示层：`NTEUID/nte_role/role_card.py` 负责把 `CharacterScore` 渲染成角色卡。

## 二、角色伤害

伤害计算追求“真实面板 × 真实倍率表 × 乘区公式”，由 `build_member_damage` / `build_character_damage` 驱动，结果用于「最强一击」「循环期望」等卡片。

### 2.1 数据来源

1. **真实面板** `parse_panel(character) -> PanelStats`
   - 从 `character.properties` 取 `atk / defense / hpmax / crit_rate(0~1) / crit_dmg(0~1) / general_dmg / element_dmg`。
   - 元素增伤只取角色自身元素那一条（`damageup<元素>base`）。
   - `base_atk`（白值）：`白值 = (面板攻 - 结构化固定攻) / (1 + 结构化攻%)`，且必须 ≥ `角色基础攻 + 武器基础攻`（反解失败则退回下界，避免高估外部攻% 增量）。白值**只用于外部攻击% 增益的精确增量**，不重复计入面板攻。
2. **技能倍率表** `load_ability_profiles(char_id)`
   - 来自资源仓库的角色技能数值模板，含随等级变化的变量；通过 `_eval_template` 按技能等级展开成 `(pct, flat, scale)`。
   - 共鸣 1「技能等级提升 N 级」会按类型（普攻/变轨技能/极轨终结/援护技）补回隐藏 `+N` 级（`_resonance_skill_bonus`）。
3. **敌人** `EnemyProfile`
   - `level`（默认 `80`）、`resist`（默认 `0.2`）、`def_reduction`（减防）、`res_reduction`（减抗）、`def_ignore`（无视防御）。
4. **增益**：全局 `BuffBundle`（atk_pct / dmg_pct / crit_rate / crit_dmg / def_ignore / res_ignore）+ 来源限定 `ParsedBuff`（按技能段匹配）。

### 2.2 单段伤害 `compute_segment`

```
scale_value = panel.atk（或 defense / hpmax，由 scale 决定）
若 scale == ATK 且 bundle 含外部 atk_pct：
    scale_value = panel.atk + panel.base_atk × bundle.atk_pct   # 白值×Δ% 的精确增量

base      = scale_value × pct / 100 + flat

增伤区 dmg_bonus = 1 + general_dmg + element_dmg + bundle.dmg_pct
防御区 def_mult  = (100 + 角色等级)
                  / ((100 + 角色等级) + (90 + 敌人等级) × (1 - 减防) × (1 - 无视防御))
抗性区 res_mult  ：
    有效抗性 = 原始抗性 - 减抗 - 无视抗
    有效抗性 ≥ 0 → 1 - 有效抗性
    有效抗性 < 0 → 1 - 有效抗性 / (1 - 有效抗性)     # 负抗收益被有理函数压低（非线性）
暴击区 ：
    期望系数 = 1 + min(1, 暴击率) × 暴击伤害
    必暴系数 = 1 + 暴击伤害

non_crit = base × dmg_bonus × def_mult × res_mult
crit      = non_crit × 必暴系数
expected  = non_crit × 期望系数
```

### 2.3 多段聚合

- 每个技能（`AbilityDamage`，类型 melee/skill/ultraskill/qte）含若干倍率段（`SegmentDamage`）。
- **倍率提升型增益**（`buff.kind == "mult_pct"`）直接乘到该段 `pct` 上（基础伤害区），按段名/描述精准匹配（如「噩梦」「兽牙影刺」「终结伤害倍率」），且“额外伤害倍率”与“普通倍率”互不串扰。
- **来源限定增益**（`scope` 非空，如「极轨终结造成的伤害 +X%」）：逐段按 `ability.type / ability.name / segment_name` 匹配后只折给对应段（`bundle_for_segment`），不平均摊到所有段。只折算 `dmg_pct / crit_rate / crit_dmg`，攻% / 无视防 / 无视抗恒为全局增益，绝不在这里重复计。
- **技能等级**：玩家面板等级 + 共鸣隐藏 `+N`。

### 2.4 循环期望 `rotation_expected`

用于“一轮输出”口径，规则（`models.py`）：

- 剔除**情境段**：下落 / 坠落 / 极限反击 / 闪避 / 瞄准。
- 普攻（melee）的**多形态连段**（近战 / 远程 / 射弹 / 强力等不同前缀）各形态都计入一轮求和（旧版只取最强一套会漏算整条形态）。
- 非连段的具名输出段（分支 / 蓄力 / 具名招）同样计入。

### 2.5 增益聚合原则（`buffs.py`）

- 全局增益求和；**无视防御 / 无视抗性取最大值而非求和**（同效果“提升至 X%”是替换升级，非叠加）。
- 元素限定增伤（`buff.element` 非空）仅在与成员同元素时计入。
- 全局 bundle 与逐段来源限定增益从**同一份** `member_buffs` 派生，避免来源错位。

### 2.6 关键常数（`constants.py`）

| 常数 | 值 | 说明 |
| --- | --- | --- |
| `CHAR_DEF_LEVEL_CONST` | `100` | 防御区角色项（恒 100） |
| `ENEMY_DEF_LEVEL_CONST` | `90` | 防御区大世界敌人项（轨外之境基线） |
| `DEFAULT_ENEMY_LEVEL` | `80` | 伤害假设的敌人等级 |
| `DEFAULT_ENEMY_RESIST` | `0.2` | 敌人全属性抗性基线（弱点 0.16 / 硬敌 0.28） |

暴击率封顶 `1.0`，抗≥100% 时不产生负伤害（免伤为 0）。

### 2.7 入口与展示

- 单一入口：`build_member_damage(character, enemy, member_buffs) -> (CharacterDamage, BuffBundle)`。
- 单角色：`build_character_damage(...)`。
- 展示层：`NTEUID/nte_role/strongest_board_card.py` 等取 `max(segment.crit)`（最强一击）或 `Σ rotation_expected`（循环期望）做排行/卡片。

## 三、相关文件路径速查

```
NTEUID/
├─ nte_role/
│  ├─ score.py                 # 评分核心（ScorePlan / score_character）
│  ├─ role_card.py             # 评分卡展示
│  └─ strongest_board_card.py  # 伤害排行/最强一击卡展示
├─ utils/
│  ├─ damage/
│  │  ├─ formula.py            # 单段伤害与防御/抗性/暴击乘区
│  │  ├─ profiles.py           # 面板解析、技能倍率展开、伤害构建
│  │  ├─ models.py             # 数据结构（PanelStats/SegmentDamage/AbilityDamage…）
│  │  ├─ buffs.py              # 增益聚合（全局/来源限定）
│  │  └─ constants.py          # 等级/抗性基线与常数
│  └─ resource/
│     └─ RESOURCE_PATH.py      # SCORING_PATH = .../resource/scoring
└─ resource/                   # 资源仓库（NteMeta）克隆而来
   └─ scoring/
      ├─ attributes.json       # 属性评分权重
      ├─ grades.json           # 评级阈值
      └─ chars/<char_id>.json  # 每角色评分方案
```

> 评分与倍率表均来自资源仓库 `NteMeta`（`SCORING_PATH` 指向 `NTEUID/resource/scoring`）。调整权重、评级或角色方案时改资源仓库对应文件即可，无需动代码。
